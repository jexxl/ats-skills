# ats-algorithm-doc-generation

生成 ATS 算法模型“原始实验运行数据及测试报告”Markdown 和 Word 文档。

## 使用方式

安装 skill 后，在对话中说“帮我生成算法测试报告”“生成原始实验运行数据及测试报告”或“把这份报告转成 Word”即可触发。Agent 会引导你完成以下步骤：收集基本信息 → 检查完整源代码、输入输出数据和真实运行证据 → 填写报告 Markdown → 生成 Word 文档。

## 依赖工具

- `uv`：用于运行 PEP 723 文档构建脚本
- `pandoc`：用于将 Markdown 文档转换为 Word

## 编号规则

算法编号采用 `课题编号-专题编号-基础(J)/专用(Z)-模型编号` 格式，例如 `4-4-Z-1`。Word 文档表格中的 `模型编号` 字段填写该算法编号。

文档文件名和文档内容均使用算法编号，不生成 Docker 打包编号、镜像名或 tar 文件。

## 交付物结构

```text
docs/
├── {算法编号}_原始实验运行数据及测试报告.md
└── {算法编号}_原始实验运行数据及测试报告.docx
```

## 人工审查

- 课题名称、专题名称、模型编号、模型名称、服务场景、接口模型编号、交付时间和责任单位是否均由用户提供或明确确认？
- 报告是否基于完整源代码、完整输入输出数据和真实运行证据填写？
- `测试数据与配置`、和 `测试结果` 是否来自真实文件和真实运行结果？
