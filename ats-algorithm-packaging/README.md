# ats-algorithm-packaging-skill

将 Python 算法模型打包为标准化 Docker 格式，用于云计算平台部署或数据汇交。

## 使用方式

安装 skill 后，在对话中说"帮我打包算法"或"把这个算法打包成 Docker"即可触发。Agent 会引导你完成以下步骤：收集基本信息 → 整理代码与数据 → 生成配置文件 → 构建并验证 Docker 镜像 → 生成测试报告 → 导出 Docker 镜像。

## 依赖工具

- `uv`：用于 Python 依赖锁定、运行 PEP 723 文档构建脚本，以及生成 `uv.lock`
- `pandoc`：用于将 Markdown 文档转换为 Word
- `docker`：用于构建镜像、验证运行和导出 `{打包编号}.tar`

## 编号规则

算法编号采用 `课题编号-专题编号-基础(J)/专用(Z)-模型编号` 格式，例如 `4-4-Z-1`。Word 文档表格中的 `模型编号` 字段填写该算法编号。

打包编号用于文件夹名、Docker 镜像名和镜像 tar 文件名，由算法编号转换得到：前缀加 `algo`，并全部小写，例如 `4-4-Z-1` 转为 `algo4-4-z-1`。

文档文件名和文档内容使用算法编号；打包文件夹、Docker 镜像名和 tar 文件使用打包编号；Docker 镜像标签固定为 `v1`。

## 交付物结构

```text
{打包编号}/
├── data/
│   ├── input/          # 运行所需的输入数据与配置文件
│   └── output/         # 算法运行后生成的输出数据
├── docs/
│   ├── {算法编号}_原始实验运行数据及测试报告.md
│   └── {算法编号}_原始实验运行数据及测试报告.docx
├── src/                # 算法源代码
├── Dockerfile
├── .dockerignore
├── params.json         # docker run 额外参数（供云平台读取）
├── pyproject.toml
├── uv.lock
└── {打包编号}.tar       # 最终交付的 Docker 镜像文件
```

## 人工审查

- 算法编号、打包编号、算法名称及各类文件命名是否符合规范且前后一致？
- `docs/` 中单份测试报告 Markdown/Word 的业务描述、输入输出字段、实际运行数据、测试结果和测试结论是否准确完整？
- `src/`、`main.py`、配置文件和依赖声明是否与实际算法一致？
- Docker 镜像是否能成功构建、运行并生成预期输出？
- 按照生成的测试说明文档改变配置文件后，算法是否能正确执行，并在大多数情况下有不同的输出结果？
